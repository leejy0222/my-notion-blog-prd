## 1. 프로젝트 개요

### 1.1 프로젝트명
개인 개발 블로그 (Personal Tech Blog with Notion CMS)

### 1.2 목적
Notion을 CMS로 활용하여 개인 기술 블로그를 구축하고, Notion에서 글을 작성하면 자동으로 블로그에 반영되는 시스템 구현

### 1.3 CMS 선택 이유
- **용이한 콘텐츠 관리**: Notion의 직관적인 UI로 글 작성 및 관리
- **자동 동기화**: API를 통한 실시간 콘텐츠 업데이트
- **구조화된 데이터**: 메타데이터(카테고리, 태그, 상태 등) 관리 용이
- **별도 백엔드 불필요**: Notion API만으로 충분
- **비용 효율성**: Notion 무료 플랜으로도 충분

### 1.4 대상 사용자
개인 개발자 (블로그 운영자 및 방문자)

## 2. 주요 기능

### 2.1 핵심 기능
1. **글 목록 페이지**
   - Notion 데이터베이스에서 발행된 글 목록 조회
   - 최신순 정렬
   - 글의 제목, 카테고리, 발행일, 요약 정보 표시

2. **글 상세 페이지**
   - 개별 글의 전체 내용 표시
   - Notion 블록 렌더링 (텍스트, 이미지, 코드 블록 등)
   - 메타데이터 표시 (작성일, 카테고리, 태그)
   - 이전/다음 글 네비게이션

3. **카테고리 필터링**
   - 카테고리별 글 목록 조회
   - 카테고리 목록 표시
   - 동적 URL 라우팅 (/category/[slug])

4. **검색 기능**
   - 제목 및 내용 기반 검색
   - 실시간 검색 결과 표시
   - 검색 결과 필터링

5. **반응형 디자인**
   - 모바일, 태블릿, 데스크톱 최적화
   - 터치 친화적 UI
   - 접근성 고려 (a11y)

### 2.2 추가 기능 (Phase 2)
- 댓글 시스템 (Utterances, Disqus 등)
- 관련 글 추천
- 조회수 트래킹
- 소셜 공유 기능
- 다크 모드
- RSS 피드 생성

## 4. Notion 데이터베이스 구조

### 4.1 메인 데이터베이스 (Blog Posts)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| Title | Title | ✓ | 블로그 글 제목 |
| Slug | Text | ✓ | URL용 슬러그 (e.g., "my-first-post") |
| Category | Select | ✓ | 카테고리 (e.g., React, Next.js, TypeScript) |
| Tags | Multi-select | | 태그 (여러 개 선택 가능) |
| Published Date | Date | ✓ | 글 발행 일자 |
| Status | Select | ✓ | 상태 (초안/발행됨/보관됨) |
| Content | Page | ✓ | 글 본문 (Notion 블록 형식) |
| Excerpt | Text | | 글 요약 (500자 이내) |
| Featured Image | Files & media | | 대표 이미지 |
| Reading Time | Number | | 예상 읽기 시간 (분) |
| View Count | Number | | 조회수 (선택사항) |

### 4.2 카테고리 설정
```
- React
- Next.js
- TypeScript
- Tailwind CSS
- 성능 최적화
- DevOps
- 기타
```

```
- Next.js 15 프로젝트 생성
- TypeScript 설정
- Tailwind CSS 설정
- shadcn/ui 통합

```

- Notion API 키 발급
